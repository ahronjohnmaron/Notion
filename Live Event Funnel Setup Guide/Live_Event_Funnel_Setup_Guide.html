Live Event Funnel Setup Guide

Live Even Funnel Full Implementation Guide

GoHighLevel | Loveable | Zoom

A complete, step-by-step guide to building and maintaining your live event funnel.

Follow the phases in order. Do not skip ahead.

FUNNEL MAP

BEFORE YOU BEGIN

⚠️ READ THIS FIRST

This guide is designed for non-technical users. Every step is written in plain language. If something is unclear, re-read it before reaching out for support — most answers are in this document.

What You Will Build

By the end of this guide, you will have a fully working live event funnel that:

  • Captures cold traffic leads via an opt-in page
  • Automatically routes warm leads directly to a booking page
  • Books 1:1 pre-event calls with automated reminders
  • Qualifies leads and books them into live events and post-event calls
  • Sends reminder and warm-up sequences via SMS and email
  • Runs repeatedly — just add new dates, no rebuilding required

Funnel Logic Map

Read this map before building. It shows the complete flow from first contact to post-event call.

ENTRY PATH 1 — COLD TRAFFIC
▼
Opt-In Page → Form (Name, Email, Phone)
▼
Redirect → Booking Page → Pre-Event Call Calendar
ENTRY PATH 2 — WARM TRAFFIC
▼
Direct Link → Booking Page → Pre-Event Call Calendar
▼
PRE-EVENT CALL (1:1) | Automated SMS + Email Reminders
▼
DECISION POINT: Is this person a fit?

NOT A FIT → Exit sequence. No further booking.

IS A FIT → Book into Live Event + Post-Event Call

▼
LIVE EVENT (Group Calendar) | SMS + Email Warm-Up
▼
POST-EVENT CALL (1:1) | Booked on Pre-Event Call
ℹ️ HOW THE DECISION POINT WORKS The decision point (fit / not fit) happens on the Pre-Event Call. Only you as the host decide — there is no automated logic for this. You manually book fits into the live event and post-event call.

1 PHASE 1: Core Setup — GHL Foundations

Account settings, custom fields, tags, and pipeline

▶ VIDEO COMING SOON / Phase 1 — GHL Core Setup Walkthrough

Step 1.1 — Account Settings

Go to: GHL > Settings > Business Profile

  1. Set your Business Name
  2. Set your Timezone (this affects all calendar and automation timing — get this right)
  3. Add your Business Phone Number
  4. Add your Business Address
  5. Add your Business Email
  6. Save changes
⚠️ WARNING

Timezone is critical. Every automation and calendar availability runs on this timezone. If it is wrong, your reminders will fire at the wrong time.

Step 1.2 — Create Required Custom Fields

Go to: GHL > Settings > Custom Fields > Add Field

Create each of these fields exactly as listed:

Field Name Field Type
cf_pre_event_call_dateDate
cf_live_event_dateDate
cf_post_event_call_dateDate
cf_fit_statusDropdown: Fit / Not a Fit / Pending
ℹ️ NOTE

These fields let you track each lead's journey. Do not rename them later — automations reference these field names.

Step 1.3 — Create Required Tags

Go to: GHL > Contacts > Tags > Create Tag

Create all of the following tags now:

  • tag-optin-submitted
  • tag-pre-event-booked
  • tag-pre-event-attended
  • tag-pre-event-no-show
  • tag-fit
  • tag-not-a-fit
  • tag-live-event-booked
  • tag-live-event-attended
  • tag-post-event-booked
  • tag-post-event-attended
  • tag-did-not-book-pre-event
⚠️ KEEP IT CLEAN

Do not create extra tags on the fly. Stick to this list. Extra tags make automations hard to debug.

Step 1.4 — Create a Simple Pipeline (Optional but Recommended)

Go to: GHL > Opportunities > Pipelines > Create Pipeline

Name it: PIPE — Live Event

Create these stages in order:

  1. New Lead
  2. Pre-Event Call Booked
  3. Qualified — Event Booked
  4. Attended Event
  5. Post-Event Call Completed
ℹ️ NOTE

The pipeline gives you a visual overview of where each lead is. It is optional — automations run via tags and triggers, not pipeline stages.

How to Add Event Dates

Instead of setting open availability, you add individual event slots:

  1. Inside the calendar settings, go to: Event Slots / Specific Dates
  2. Click Add Date / Add Slot
  3. Enter the date and start time of your event
  4. Save
  5. Repeat for each upcoming event date
✅ ROLLING EVENT SYSTEM

IMPORTANT: You never need to rebuild this calendar. When you run a new event, just come back here and add a new date slot. The calendar URL stays the same. Your automations keep working. If your event is over two days just make sure to add both dates.

Managing Multiple Event Dates

  • You can have several upcoming dates visible at once
  • Only book people into your upcoming event unless they specifically need the next
  • Once an event date is full (max attendees reached), it closes automatically
  • Past event dates disappear from the booking page automatically

What NOT to Touch After Booking

⚠️ WARNING — DO NOT DELETE BOOKED DATES

Once leads have booked into an event date, do NOT delete that date or change the event time. This breaks their booking and Zoom link. If you need to cancel, notify all attendees manually first.

3 PHASE 3: Forms & Data Capture

GHL opt-in form with tagging and redirect

▶ VIDEO COMING SOON / Phase 3 — Form Build & Configuration

Step 3.1 — Build Your Opt-In Form

Go to: GHL > Sites > Forms > New Form

  1. Name your form: FORM — Opt-In
  2. Add the following fields:
    • First Name (required)
    • Last Name (required)
    • Email Address (required)
    • Phone Number (required)
  3. Add an SMS Consent Checkbox. Label it exactly:
    "By checking this box, you agree to receive SMS messages from [Your Business Name]. Message and data rates may apply. Reply STOP to unsubscribe."
  4. Map the checkbox to the custom field: cf_sms_consent
  5. Set button text to: REGISTER NOW (or your preferred CTA)
⚠️ LEGAL REQUIREMENT

The SMS consent checkbox is legally required if you plan to send marketing SMS. Do not skip it.

Step 3.2 — Set Up Redirect

  1. Inside the form settings, go to: Redirect After Submit
  2. Enter the URL of your Booking Page (you will build this in Phase 5)
  3. For now, you can leave this blank and return to fill it in after Phase 5
ℹ️ HOW THIS CONNECTS

The form redirect sends the lead straight to the calendar booking page after they opt in. This is how Entry Path 1 connects the opt-in to the pre-event call booking.

4 PHASE 4: Automations

Three clean, linear workflows

ℹ️ KEEP IT SIMPLE

Build ONLY the 4 workflows listed here. Do not add extras. Simple automations are easier to debug. Each workflow has one job.

▶ VIDEO COMING SOON / Phase 4 — Building the 3 Automations

Workflow A — Pre-Event Call Reminders

Name: WF — Pre-Event Booked — Reminders

Trigger: Calendar Appointment — CAL — 1:1 — Pre-Event Call — Appointment Booked

Actions (in order)

  1. Add Tag: tag-pre-event-booked
  2. Send Email: Booking Confirmation — include date, time, Zoom link, and what to prepare
  3. Send SMS: Booking confirmed! Here is your call link: {{zoom_link}}. See you soon!
  4. Wait: Until 24 hours before appointment
  5. Send Email: 24-hour reminder — include Zoom link
  6. Send SMS: Reminder — your call is tomorrow at {{appointment_time}}. Link: {{zoom_link}}
  7. Wait: Until 1 hour before appointment
  8. Send SMS: Your call starts in 1 hour. Link: {{zoom_link}}
  9. Wait: Until 5 minutes before appointment
  10. Send Email & SMS
ℹ️ MERGE FIELDS TIP

Use GHL merge fields: {{contact.first_name}}, {{appointment.start_time}}, {{appointment.zoom_join_url}}. Your GHL account will have the exact merge field names — check under the merge field button inside the email/SMS editor.

Workflow B — Live Event Warm-Up Sequence

Name: WF — Live Event Booked — Warmup + Reminders

Trigger: Calendar Appointment — CAL — GROUP — Live Event — Appointment Booked

Actions (in order)

  1. Add Tag: tag-live-event-booked
  2. Update Pipeline stage (If you are doing this)
  3. Create your event reminder emails & sms using the GPT Prompt
  4. Load them into your workflow using the appropriate wait times between each

Workflow C — Post-Event Call Reminders

Name: WF — Post-Event Booked — Reminders

Trigger: Customer Appointment Booked — CAL — 1:1 — Post-Event Call — Appointment Booked

Actions (in order)

  1. Add Tag: tag-post-event-booked
  2. Send Email: Booking confirmation with Zoom link and call agenda
  3. Wait: 1 Day Before Appt
  4. Send SMS & Email
  5. Wait: 1 hour before appointment
  6. Send SMS & Email
  7. Wait: 5 Minutes before appointment
  8. Send SMS & Email

Workflow D — Opted In - Did Not Book Pre-Event

Name: WF — Opted in — Did Not Book Pre-Event Call

Trigger: Form submitted — Filter: Form is — FORM — Opt-In

Actions (in order)

  1. Wait 30 minutes
  2. If/Else: Branch 1: tag does not include tag-pre-event-booked, Branch 2: tag includes tag-pre-event-booked
  3. Branch 1: email, sms, wait 2 hours / Branch 2: empty
  4. If/Else: Branch 1: tag does not include tag-pre-event-booked, Branch 2: tag includes tag-pre-event-booked
  5. Branch 1: email, sms, wait 2 hours / Branch 2: empty
  6. If/Else: Branch 1: tag does not include tag-pre-event-booked, Branch 2: tag includes tag-pre-event-booked
  7. Branch 1: add tag tag-did-not-book-pre-event / Branch 2: empty
⚠️ DO NOT ADD MORE UNTIL TESTED

These 4 workflows cover the entire funnel. Do not build additional reminders or sequences until these are fully tested (Phase 7).

5 PHASE 5: Page Setup — Loveable

Opt-in page and booking page

▶ VIDEO COMING SOON / Phase 5 — Building Pages in Loveable

Step 5.1 — Opt-In Page

Your prompts and copy are already provided. Use them exactly as given.

  1. Log into Loveable
  2. Create a new page: Opt-In Page
  3. Paste your provided prompt into Loveable to generate the page structure
  4. Replace any placeholder text with your provided copy
  5. Embed your GHL form:
    • In GHL, go to: Sites > Forms > FORM — Opt-In > Embed
    • Copy the embed code
    • In Loveable, add an Embed / Code Block element where the form should appear
    • Paste the GHL form embed code
  6. Check mobile view and ensure form is readable and buttons are tappable
  7. Publish the page and copy the URL

Step 5.2 — Booking Page

The booking page is where leads go after opting in (Entry Path 1) or directly for warm traffic (Entry Path 2).

  1. Create a new page in Loveable: Booking Page
  2. Use your provided prompt and copy
  3. Embed your GHL Pre-Event Call Calendar:
    • In GHL, go to: Calendars > CAL — 1:1 — Pre-Event Call > Embed Calendar
    • Copy the embed code
    • In Loveable, add an Embed / Code Block where the calendar should appear
    • Paste the GHL calendar embed code
  4. Check mobile view — the calendar must scroll properly on phones
  5. Publish the page and copy the URL

Step 5.3 — Final Connection

  1. Go back to GHL > Sites > Forms > FORM — Opt-In
  2. In the Redirect After Submit field, paste your Booking Page URL
  3. Save
✅ ENTRY PATH 1 CONNECTED

After this step, Entry Path 1 is fully connected: Opt-in form submit → redirect to booking page → lead books a pre-event call

6 PHASE 6: Domain Setup

Connect your custom domain to Loveable

▶ VIDEO COMING SOON / Phase 6 — Domain & DNS Setup

Step 6.1 — Add Your Domain in Loveable

  1. Go to Loveable > Settings > Custom Domain
  2. Enter your domain name (e.g., events.yourbusiness.com)
  3. Loveable will show you DNS records to add — keep this screen open

Step 6.2 — Add DNS Records at Your Domain Registrar

Go to where you bought your domain (e.g., GoDaddy, Namecheap, Google Domains):

  1. Log in and find: DNS Settings / DNS Records / Manage DNS
  2. Loveable will give you a CNAME record. It will look like:
DNS Field What to Enter
TypeCNAME
Name / Hostevents (or whatever subdomain you choose)
Value / Points To[The value Loveable shows you — paste exactly]
TTL3600 (or Auto)
  1. Save the DNS record
  2. DNS changes take 15 minutes to 24 hours to go live. This is normal.

Step 6.3 — Verify SSL Certificate

  1. Return to Loveable > Settings > Custom Domain
  2. Wait for the status to show: Active or Verified
  3. Visit your domain in a browser — you should see a padlock icon (https://)
ℹ️ SSL NOTE

If your domain shows Not Secure or a warning, wait a few more hours and check again. If it still fails after 24 hours, re-check the DNS record was entered correctly.

How to Test Like a Real Lead

Use a personal email address and a real phone number (not your business ones). Walk through the entire funnel as if you are a new lead discovering you for the first time.

Entry Path 1 — Cold Traffic Test

  • Visit your opt-in page URL — does it load correctly on desktop and mobile?
  • Fill in the form with your real name, email, and phone number
  • Check the SMS consent box
  • Submit the form
  • Were you redirected to the booking page?
  • Did the tag tag-optin-submitted appear on your contact in GHL?
  • Book a pre-event call on the calendar
  • Did you receive a confirmation email with a Zoom link?
  • Did you receive a confirmation SMS?
  • Did the tag tag-pre-event-booked appear on your contact?
  • Check GHL > Calendars — does the appointment appear?
  • Click the Zoom link — does it open Zoom correctly?

Entry Path 2 — Warm Traffic Test

  • Visit your booking page URL directly
  • Book a pre-event call
  • Same checks as above from 'Did you receive a confirmation email...' onward

Live Event Calendar Test

  • Go to GHL > Calendars > CAL — GROUP — Live Event
  • Add a test event date (tomorrow or next week)
  • Book into the test event using your test contact
  • Did you receive a confirmation email with the event Zoom link?
  • Did you receive a confirmation SMS?
  • Did the tag tag-live-event-booked appear?

Post-Event Call Calendar Test

  • Book a post-event call using your test contact
  • Did you receive a confirmation email with a Zoom link?
  • Did you receive a confirmation SMS?
  • Did the tag tag-post-event-booked appear?

Automation Timing Test

  • Book a test appointment within 24 hours from now
  • Wait — did the 24-hour reminder email arrive at the right time?
  • Wait — did the 24-hour reminder SMS arrive?
  • Book a test appointment within 1 hour from now
  • Did the 1-hour reminder SMS arrive?

8 PHASE 8: Resetting & Maintaining the Funnel

How to run this system long-term

📌 CRITICAL — READ THIS

This phase is just as important as building the funnel. Read it carefully so you know how to maintain and reuse the system without breaking it.

▶ VIDEO COMING SOON / Phase 8 — Maintaining Your Funnel Long-Term

How to Add New Event Dates

This is the ONLY thing you need to do between events:

  1. Go to GHL > Calendars > CAL — GROUP — Live Event
  2. Go to Event Slots / Specific Dates
  3. Click Add New Date
  4. Enter your new event date and time
  5. Set max attendees
  6. Save

That is it. Your calendar URL, automations, and pages all remain the same. Nothing else needs to change.

What NOT to Touch

⚠️ DO NOT TOUCH

Do NOT delete or rename your calendars, workflows, forms, or tags after the funnel is live. Other parts of the system are connected to them by name. Changing them breaks the connections.

Item Can You Edit It?
Calendar namesNO — breaks automations
Workflow trigger settingsNO — only edit content inside actions
Tag namesNO — breaks trigger logic
Custom field namesNO — breaks form mappings
Form field namesOnly if you know all places they are referenced
Email / SMS contentYES — safe to update anytime
Event dates (adding new)YES — this is how you reuse the system
Booking page copy in LoveableYES — safe to update
Calendar availability hoursYES — but test after

How to Reuse Automations Safely

  • Your 3 workflows run forever — they do not need to be reset between events
  • They fire based on calendar booking triggers, so they automatically apply to new events
  • Do not duplicate workflows — update the email / SMS content inside the existing ones
  • If you want to change reminder timing, edit the Wait steps inside the existing workflow

How to Avoid Duplicate Workflows

  • Before creating any new automation, check the existing workflow list first
  • If the trigger already exists in a workflow, edit that workflow — do not create a new one
  • Name every workflow using the naming convention: WF — [Trigger] — [Action]

How to Keep the System Clean Long-Term

  1. Every quarter: review your contact tags and archive leads who are more than 90 days old with no activity
  2. Every event cycle: delete the test event date you created during Phase 7 testing
  3. If you onboard a new team member: have them read this entire guide before touching anything
  4. Do not install GHL apps or third-party integrations without checking if they conflict with Zoom
